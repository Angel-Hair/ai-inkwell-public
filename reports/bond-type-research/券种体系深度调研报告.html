📋 券种体系调研报告

算力券 · Token券 · 场景券 —— 券种体系深度调研报告

算力超市 · Token 工厂 · 券种体系 —— 概念拆解，横向对比，落地逻辑

2026 年 6 月基于多源政府文件与权威媒体报道交叉验证

引言

2023 年以来，伴随"东数西算"工程深化与大模型产业爆发，中国 AI 基础设施领域涌现出一系列新概念——算力超市、Token 工厂、算力券、Token 券、场景券。这些术语散见于政府文件、企业宣传与媒体报道，彼此交叉重叠，极易混淆。

本报告基于国务院及工信部文件、15 个以上省份/城市的政策原文、运营商与云厂商的公开商业数据，对五个核心概念逐一拆解，明确区分三类性质——营销概念（企业包装的商业话语）、国家计划（有正式文件背书的政策工具）、落地应用（已上线运营或完成兑付的实际产品）。

核心发现：算力券是最成熟的政策工具，已覆盖全国 15+ 城市/省份；Token 工厂从行业术语快速演变为百亿级产业实践；四川于 2026 年 5 月首次在省级文件中提出"词元（Token）券"；场景券正从概念走向独立财政管理——山东已发布专项资金管理办法征求意见稿。与此同时，多个概念面临商业主体借政策话语进行营销包装的风险，本报告专设一章标定边界。

一、五大概念逐个拆解

1. 算力超市落地应用国家政策概念
"类似电商平台的算力交易市场，支持按'卡时''核时'及 Token 计费等灵活付费，用户可直接登录平台选择不同算力产品，像购买日用品一样完成支付并使用。" —— 工信部，2026.4.15

运作模式：供给侧通过"算力银行"存入闲置算力→池化调度→跨区域调配；需求侧在"算力超市"浏览→按卡时/Token计费→在线下单。核心逻辑：算力从重资产变为可交易、可计量的轻服务。工信部 2026 年 4 月首次在官方文件中提出此概念，目标到 2028 年底建成全国普惠算力服务体系。

3. 算力券国家计划最成熟

地方政府向 AI 企业发放的算力消费补贴代金券，用于抵扣指定智算中心的 GPU/NPU 算力费用。是"创新券"在 AI 领域的延伸，也是"东数西算"国家战略的核心配套。

4. Token 券（词元券）进入官方政策早期阶段
"修订省促进人工智能产业链发展若干政策，扩容'算力券'、启动'词元（Token）券'" —— 四川省人民政府办公厅，川办发〔2026〕14号，2026.5.5

这是首次在省级政府正式文件中出现 Token 券提法（四川采用"词元券"中文命名），标志着 Token 券从商业营销概念进入官方政策工具范畴。目前仍处于"启动"阶段，具体实施细则尚未公布。

此外，贵州算力券已覆盖"模型服务"（30% 补贴，上限 200 万/年），功能上与 Token 券高度重叠。商业层面，中国移动北京公司已推出"算力 Token 套餐"，以 Token 计费、按需付费。

⚠️ 核心区分：商业促销 vs 政府补贴
维度商业"Token 券"政府"词元（Token）券"
来源云厂商（阿里、百度、讯飞等）促销活动四川川办发〔2026〕14号正式提出
性质企业营销行为，预付费套餐的"代金券"包装财政补贴工具（细则未出，已入政府公文）
资金企业营销预算财政资金（预计走产业链专项资金）
渠道云厂商官网 / 控制台待公布（预计类似算力券的政府平台模式）

📌 实用判断标准：看发放主体是云厂商还是政府，看申领渠道是企业官网还是政府平台。云厂商的"Token 券"本质是打折促销。目前仅四川在政府公文中提出"词元券"，但兑付通道尚未建立——任何声称"响应国家政策发放 Token 券"的商业行为当前均为借势营销。

🔑 Token 券正从商业概念进入政策视野。需严格区分云厂商的商业促销与政府的财政补贴。
5. 场景券国家计划部分省试点

场景券是真实存在的政策工具，多份省级文件明确提及，且正快速走向独立财政管理。

文件核心原文时间

概念溯源：从"场景培育"到"场景券化"

国务院层面（2025.11）：国办发〔2025〕37号将场景定义为"用于系统性验证新技术、新产品、新业态产业化应用的具体情境"，政策工具为政府采购、基建投资、首台套支持——本质是供给侧开放资源，政府作为场景提供方。全文无"券"字，无直接发钱机制。

地方层面（2026.2 起）：四川率先"探索制定"，山东更进一步——2026.5.21 宣布百亿计划，6.10 即发布独立资金管理办法征求意见稿，表明场景券正获得独立财政科目。

核心张力：国务院的"场景"是开放资源；地方的"场景券"是发放补贴。算力券有清晰的"合同→发票→兑付"闭环。场景券的财政路径更可能走政府采购通道或项目制补贴，而非独立的"发券-核销"体系。

🔑 场景券真实存在且正快速制度化。与算力券的"消费品补贴"逻辑不同，它更接近"项目制/采购制"逻辑。

四、⚠️ 商业营销与政府政策的边界警示

多个概念存在商业主体借政府政策话语进行营销包装的现象，以下逐一标定边界。

🔴 高风险混淆区

Token 券 / 词元券 —— 最易混淆

云厂商的"Token 券"（2024–2025 年已常见）是预付费套餐的促销包装。政府的"词元券"（四川 2026.5 首次提出）是财政补贴工具，目前尚无兑付通道。识别方法：看发放方是否为云厂商、是否需要充值/消费后才"送"、是否通过政府平台申领。⚠️ 任何声称可"代申领政府 Token 券"的当前均为虚假宣传。

算力超市 —— 概念借用

工信部 2026 年 4 月才正式提出此概念。此前市场上已存在的商业"算力超市"平台是独立商业行为，与政府普惠算力体系无隶属关系。判断标准：是否接入地方政府算力调度平台？是否支持算力券兑付？

Token 工厂 —— 无政府背书

此概念从未在任何政府文件中出现。运营商（电信/移动/联通）的 Token 工厂集采是企业商业行为，使用企业自有资金，非财政拨款。部分媒体或券商将其包装为"国家战略"属于过度解读。

🟡 中度混淆区

场景券 —— 政策概念被泛化

国务院只提"场景培育"（无"券"字）。地方"场景券"是自行"券化"的产物。参与政府"场景培育"项目 ≠ 获得了"场景券"。目前仅山东（征求意见稿）、四川（探索）、杭州萧山（已发布）有明确动作。

模型券 —— 与算力券易混淆

算力券补"买算力"，模型券补"买模型"。但贵州已将"模型服务"纳入算力券覆盖范围（30%），在贵州二者实际合一。其他省份分列，申领时需注意避免重复申报。

🟢 清晰区
  • 算力券：最透明。申领渠道均为政府平台（.gov.cn），有合同→发票→兑付闭环。注意：须本地注册、须使用指定智算中心、须未获其他同类补贴。
  • 语料券 / 存力券 / 运力券：目前仅有政策宣示，尚无大规模落地，暂未见商业借势营销。
🔑 三要素判断法：① 发放主体是否为政府部门 → ② 申领渠道是否为政府平台（.gov.cn）→ ③ 是否需要财政票据（发票+支付凭证）。缺一即非政府补贴。

6.3 决策逻辑

第一步：确定核心需求
├─ 需要 GPU 算力 → 算力券，选补贴最高省份（贵州 30% 或武汉 50%）
├─ 需要买/微调模型 → 模型券，山东或四川
├─ 需要部署实体场景 → 场景券，山东（制造）或杭州萧山（综合）
├─ 需要采购训练数据 → 语料券，贵州（数据交易所）或杭州萧山
├─ 需要 API 调用 → 词元券（等四川细则）或商业 Token 包
└─ 仅需弹性算力 → 算力超市，商业比价采购

第二步：选择省份（叠加策略）
├─ 单一需求 → 选该券种补贴最高省份
├─ 多需求 → 优先贵州（五合一）或山东（双券+百亿资金）
└─ 跨省布局 → 研发放高补贴省（贵州/庆阳），部署放场景券省（山东/杭州）

第三步：验证合规
├─ 在当地注册？← 几乎所有算力券的前置条件
├─ 算力在指定区域？← 贵州要求物理位置在省内
├─ 未获其他同类补贴？← 各地均禁止重复享受
└─ 合同/发票/支付凭证完备？← 政府券种均需"先付后兑"
