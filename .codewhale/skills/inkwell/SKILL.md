# Inkwell

inkwell 文档工作区的项目级 Skill。这不是代码项目——所有操作围绕 HTML / Markdown 研究报告展开。

## 工作流

### 1. 新建研究

用户要求开始新研究时触发。

创建 `reports/<name>/` 目录，内含三个文件：
- `<name>.html` — HTML 骨架，包含项目标准样式（暗色主题、JS 动态侧边目录、内联 CSS）
- `<name>-sanitized.html` — 正文初始副本，作为脱敏基线
- `AGENTS.md` — 主题、用户需求、格式要求、约束等

### 2. 脱敏处理

用户要求对报告脱敏时触发。

**原则：** 凡是能识别具体的人、机器、账号、组织的信息，一律替换为描述性占位符（如 `<XX服务器地址>`、`<用户目录>`、`<已脱敏>`、`<内网域名>`、`<邮箱地址>`、`<个人姓名>` 等），需要根据报告实际内容判断哪些信息属于可识别范围。

脱敏需要先生成副本（`<name>-sanitized.<ext>`），然后在副本上进行修改，不修改原文。
修改内容过多可以尝试在 `tmp` 目录下编写python脚本。
完成后 diff 并列出替换清单，可疑但未替换的条目标记出来即可。

### 3. Git 提交标准检查

用户要求提交时触发。通过 `git diff --cached` 检查本次提交涉及的报告，
列出缺失脱敏版本的文件后先请用户确认后再正式提交。

## 安全要求

- 脱敏时始终在副本上操作，绝不修改原始报告。
- Git 操作时绝不 force-push 或绕过 hooks。
- 此 Skill 仅限 inkwell 工作区使用，不要在其他工作区激活。
