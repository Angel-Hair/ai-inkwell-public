# Inkwell

inkwell 文档工作区的项目级 Skill。这不是代码项目——所有操作围绕 HTML / Markdown 研究报告展开。

## 工作流

### 1. 新建研究

用户要求开始新研究时触发。

创建 `reports/<name>/` 目录，内含以下文件：
- `<name>.html` 或 `<name>.md` — 报告正文（格式由内容决定）
- `<name>-sanitized.html` 或 `<name>-sanitized.md` — 脱敏基线副本
- `AGENTS.md` — 主题、用户需求、格式要求、约束等

### 2. 脱敏处理

用户要求对报告脱敏时触发。

**原则：** 凡是能识别具体的人、机器、账号、组织的信息，一律替换为描述性占位符（如 `<XX服务器地址>`、`<用户目录>`、`<已脱敏>`、`<内网域名>`、`<邮箱地址>`、`<个人姓名>` 等），需要根据报告实际内容判断哪些信息属于可识别范围。

1. 生成脱敏副本（`<name>-sanitized.<ext>`），在副本上修改，**绝不修改原文**
2. 修改内容过多时可在 `tmp/` 下编写 Python 脚本批量处理
3. 完成后 `diff` 原文与脱敏版，列出替换清单
4. 可疑但未替换的条目标注出来，交用户判断

### 3. Git 提交标准检查

用户要求提交时触发。

1. 先将新文件 `git add` 到暂存区
2. 通过 `git diff --cached` 检查本次提交涉及的报告
3. 列出缺少脱敏版本的文件（已在 `AGENTS.md` 声明豁免的除外）
4. 请用户确认后执行 `git commit` 和 `git push`

## 安全要求

- 脱敏时始终在副本上操作，绝不修改原始报告
- 绝不 force-push 或绕过 hooks（公开仓镜像由 CI 自动同步，Agent 不干预）
- 此 Skill 仅限 inkwell 工作区使用，不要在其他工作区激活
